## How to Contribute

### Reporting Bugs

If you find a bug, please open an [issue](https://github.com/PauBenetPrat/volleyflow/issues) with:

- **Clear title**: Brief description of the bug
- **Description**: What happened vs what you expected
- **Steps to reproduce**: How to trigger the bug
- **Environment**: Browser, device, OS (if relevant)
- **Screenshots**: If applicable

### Suggesting Features

Have an idea? Open an [issue](https://github.com/PauBenetPrat/volleyflow/issues) with:

- **Clear title**: Brief description of the feature
- **Description**: What the feature would do
- **Use case**: Why this feature would be useful
- **Mockups/Sketches**: If you have any

### Submitting Code Changes

1. **Fork the repository**
2. **Create a branch**: `git checkout -b feature/your-feature-name` or `fix/your-bug-fix`
3. **Make your changes**: Follow the existing code style
4. **Test your changes**: Make sure everything works
5. **Commit**: Write clear commit messages
6. **Push**: `git push origin your-branch-name`
7. **Open a Pull Request**: Describe your changes clearly

### Code Style

- Follow Flutter/Dart style guidelines
- Use meaningful variable and function names
- Add comments for complex logic
- Keep functions focused and small
- Write tests when possible

### Development Setup

1. Clone the repository:
   ```bash
   git clone https://github.com/PauBenetPrat/volleyflow.git
   cd volleyflow
   ```

2. Install dependencies:
   ```bash
   flutter pub get
   ```

3. Run the app:
   ```bash
   flutter run
   ```
